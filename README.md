## 1. 📥 `10_fetch_entrez_metadata.py`

### 【概要】

指定されたディレクトリ内のMDファイルから論文情報をスキャンし、PubMed APIから「MeSHターム」「著者キーワード」「論文種別（PublicationType）」を取得して、ローカルキャッシュ（`entrez_metadata.json`）を作成するスクリプトです。

### 💡 主な機能

1. **MDファイルスキャン & DOI抽出**

   * MDファイルのYAMLフロントマターから `citekey` と `doi` を読み込みます。
   * フロントマターに `doi` がない場合、本文内の `>[!Info]` ブロックからフォールバック抽出を試みます。

2. **キャッシュ高速化**
   * 既に `entrez_metadata.json` にデータがあり、かつDOIに変更がない論文はAPIリクエストをスキップします。

3. **PubMed API連携 (NCBI Entrez)**

   * **Step 1 (esearch)**: DOIから `PMID`（PubMed ID）を検索・取得します。
   * **Step 2 (efetch)**: XML形式の書誌情報を取得し、MeSHターム、著者キーワード、論文種別を抽出します。

4. **公式基準によるレビュー論文（Review）自動判定**
   * PubMedの `PublicationType` に `Review`, `Systematic Review`, `Meta-Analysis` のいずれかが含まれている場合、自動的に `entrez_is_review: True` と判定します。**（※手動の `tags` 判定は完全に廃止）**

5. **例外・未登録論文の可視化**
   * 処理の最後に、\*\*「PubMed未登録（書籍の章など）または取得に失敗した論文」\*\*をターミナルに一覧表示します。

***

## 2. ✍️ `20_update_md_properties.py`

### 【概要】

`10_fetch_entrez_metadata.py` が出力した `entrez_metadata.json` をデータソースとし、ローカルのMDファイルのフロントマターおよび本文の内容を自動で更新・補正するスクリプトです。

### 💡 主な機能

1. **フロントマター（プロパティ）の自動同期**

   * JSONから取得した `doi`, `pmid`, `mesh_terms`, `keywords` をフロントマターに書き込みます。
   * PubMedの判定に基づき、`review: true` または `review: false` を明示的に追加します。
   * ※PubMed未登録等でデータがない論文には、安全策として `review: false` を書き込みます。

2. **## Introduction の自動付与**
   * `# 4 Main Text` の直後に、最初の `##` ヘッダー（見出し）より前に本文が存在する場合、自動的に `## Introduction` ヘッダーを挿入して文章構造を標準化します。

3. **## Keywords セクションの挿入**
   * PubMedからキーワードが取得でき、かつ本文中に `## Keywords` セクションがまだ存在しない場合、`References` や `Acknowledgements` の直前（なければMain Textの末尾）にキーワード一覧を自動挿入します。

4. **未処理論文の可視化**
   * JSONにデータが存在しなかった（またはcitekey不一致などで紐付けられなかった）論文を、最後にターミナルに一覧表示します。

## 30_batch_convert_articles.py の機能を簡単に要約します。

---

### **スクリプトの機能要約**

このスクリプトは、**前の工程で整形された複数の論文 Markdown（MD）ファイルを読み込み、AI（LlamaIndexなど）が読み込みやすいように「構造化された1つの巨大なJSONファイル（`articles_all.json`）」に一括変換するツール**です。

主な処理は以下の3点です。

1. **論文データの構造化（パース）**

   * 各MDファイルから、タイトル、著者、出版年、ジャーナル名、DOI、PMID、MeSH、キーワードなどの「メタデータ」を抽出します。
   * 本文（`# 4 Main Text`）を解析し、`## Introduction` や `## Results` などの**見出し（セクション）ごとに文章を分割してデータ化**します。

2. **レビュー論文の判定と処理の最適化**

   * `entrez_metadata.json` から「レビュー論文か否か（`is_review`）」の判定を読み込みます（ない場合はMDのタグから判定）。
   * **レビュー論文の場合：** 構造が標準的ではないため、参考文献等を除き、本文セクションをすべて `review` タイプとして一括処理します。
   * **オリジナル論文の場合：** 本文の見出しから `intro`, `materials|methods`, `results`, `discussion` などに細かく分類します。

3. **不要な情報のクリーニングと統計出力**

   * AIの学習や検索に不要なセクション（参考文献リスト、謝辞、URLリンク、HTMLタグなど）を自動で削除し、データを軽量化します。
   * 処理完了後、変換された論文数、セクション数の分布、出版年の分布などの統計情報を画面に表示します。

 ## 📄 `40_build_all_articles_index.py` 仕様・実行説明書

このスクリプトは、一括変換された構造化JSONファイル（`articles_all3.json`）から、**LlamaIndex** を用いて高精度なセクション/段落単位のベクトルインデックス（Vector Index）を構築し、ローカル（`storage_all/`）に永続化保存するためのプログラムです。

***

## 1. 🛠️ スクリプトの主な役割と特徴

### ① セクション/段落単位のインデックス（高精度な検索の実現）

論文全体を1つの大きなテキストとして埋め込むのではなく、**「セクション」や「段落（改行 `\n\n` 区切り）」単位で細分化して `Document` オブジェクトを生成**します。 これにより、LLMが回答を生成する際に、論文内のピンポイントな記述（例：特定の実験結果や考察の1段落）を正確に検索（Retrieve）できるようになります。

### ② 論文タイプ（Regular / Review）に応じた柔軟なパース

* **一般論文（Regular）**: セクション（h2）およびサブセクション（h3）の構造を維持し、それぞれのコンテンツを段落単位に分割してインデックス化します。
* **レビュー論文（Review）**: h3サブセクションの境界を越えて、セクション全体の文脈を維持したまま段落単位に分割してインデックス化します。

### ③ 豊富なメタデータの埋め込み（ハイブリッド検索・フィルタリング用）

各段落のベクトルデータには、検索時に強力な絞り込み（メタデータ・フィルタリング）が行えるよう、以下の情報が自動的に付与されます。

* **文献基本情報**: `citekey`, `title`, `authors`, `published` (出版年), `source` (ジャーナル名), `doi`, `pmid`, `citation` (例: "Sato 2014")
* **構造情報**: `section` (セクション名), `section_type` (分類), `subsection` (サブセクション名), `paragraph_index` (段落番号), `article_type` (regular/review)
* **医学・生物学メタデータ**: `mesh_terms` (MeSHターム), `keywords` (キーワード)

## 1. `ref_formatting1.py`（参考文献リストの段落整形）

### 📌 概要

1行にまとまってしまっている、あるいは改行が崩れている `## References` セクションの文献リストを、**1文献ずつ改行し、`1. ``2. `... と番号付きリスト（Numbered List）に整形**します。

### ✨ 主な機能と特徴

* **著者名の太字を検出**: 文献の区切りを `**著者名**`（例: `**Achilli TM**`）のパターンから自動で判別します。
* **数字の太字を無視**: 巻数やページ数などの数字の太字（例: `**12**` や `**35**`）には反応しないよう、アルファベットから始まる太字のみを区切りとして認識します。
* **重複防止**: すでに `1. `などの番号が振られている場合は、二重に番号が付かないよう自動でクリーニングします。

### 🛠️ 設定方法（ファイルの指定）

スクリプト最下部の `target_file` に、処理したい Markdown ファイルのフルパスを指定します。


## 2. `remove_urls.py`（不要URL・空リンクのクリーンアップ）

### 📌 概要

Markdown ファイル内から、不要な `http://` や `https://` で始まる URL リンクを削除します。さらに、URL を削除した結果として残る**中身の空っぽなマークダウンリンク（`[]()`）を完全に消去**してファイルを美しく保ちます。

### ✨ 主な機能と特徴

* **特定のURLを保護（例外処理）**: `https://ars.` で始まる URL（例: 画像や特定のデータベースリンク）は**削除せずにそのまま残します**。

* **不要な空リンクの完全消去**: URL 削除後に残る以下の空リンクを、前後のタブ（`\t`）やスペースごと完全に消去します。

  * `[Google Scholar]()`
  * `[Crossref]()`
  * `[View in Scopus]()`
  * `[View PDF]()`
  * `[View article]()`

* **安心のログ機能**: 削除したすべての URL は、スクリプトと同一ディレクトリに **`deleted_urls.txt`** として書き出されます（万が一、後からURLを確認したくなった場合も安心です）。

* **状態に依存しない実行**: すでに手動や別処理で URL だけが消えており、空リンク（例: `[Google Scholar]()`）だけが残っている状態から実行しても、クリーンアップ（お掃除）が正しく機能します。

### 🛠️ 設定方法（ファイルの指定）

スクリプト最下部の `target_file` に、処理したい Markdown ファイルのフルパスを指定します。

```
コピー
if __name__ == "__main__":
    target_file = Path(r"C:\Users\a2189\Dropbox\obsidian\10_article\RXFP1\Alam2023.md")
    remove_urls_from_md(target_file)
```

### 🚀 実行方法

ターミナル（コマンドプロンプトや PowerShell）を開き、`uv` 環境などを利用して実行します。

```
コピー
# 1. 参考文献を1行ずつに分割して番号付きリストにする
uv run ref_formatting1.py

# 2. 不要なURLや空リンクをお掃除する

```
